# MCP 是什么？30分钟让你的 Agent 能读文件、查数据、调用一切

**作者：AI解压铺（整理：Kaze Cinder）| 发布于 2026-06-17**

---

上一期我们搭了一个能搜索和发邮件的 Agent。今天给它装上"超能力"——让它能读你的文件、查数据库、调用任何 API。

> 上期我们搭了一个能用的 Agent——会搜新闻、会写邮件。但说实话，那个 Agent 的能力很有限：它只能用我们硬编码给它的工具。
>
> 如果我说：**让你的 Agent 能读取你电脑上的任意文件、能访问你的数据库、能调用任何网站的 API**，你信不信？
>
> 这就是 **MCP（Model Context Protocol）** 做的事。它不是一个新的 AI 模型，也不是一个框架——它是一套 **让 AI 能连接一切的标准协议**。今天 30 分钟，让你亲手体验。

---

## 1. MCP 到底是什么？一个类比你就懂了

**MCP = AI 世界的万能插头。以前每接一种工具都要单独写代码，现在插上就能用。**

### 没有 MCP 的世界：

```
你要让 Agent 读文件 → 自己写代码调文件 API
你要让 Agent 查数据库 → 自己写代码调数据库 API
你要让 Agent 搜网页 → 自己写代码调网页 API
```
每个工具都要单独对接，开发成本极高。

### 有 MCP 的世界：

```
你告诉 Agent："去这个 MCP 服务器上找你需要的工具"
Agent 自动找到可用的工具，自动连接，自动使用。
你只需要装一个 MCP 服务器，所有工具一通百通。
```

### 🔍 类比

没有 MCP 之前，每个 AI 工具就像 **不同品牌的充电器**——iPhone 用 Lightning、安卓用 Type-C、老手机用 Micro-USB。你要备三根线。

**MCP 就是统一成 Type-C**——从此不管什么工具，一根线全搞定。

> **一句话：MCP = 让 Agent 能统一调用所有工具的标准协议。装一个，通所有。**

---

## 2. 实际操作：给你的 Agent 装一个 MCP 服务器

**动手时间。30 分钟，让你的 Agent 能读你电脑上的任意文件。**

### 第 1 步：安装 MCP 服务器（5 分钟）

```bash
# 安装官方 MCP 文件服务器
npx @anthropic/mcp-server-filesystem /path/to/your/files

# 启动后，MCP 服务器会在本地运行
# 它会告诉 Agent：我能读这些文件夹里的所有文件
```

### 第 2 步：把 Agent 连接到 MCP（5 分钟）

在 Agent 的配置里加上 MCP 服务器的地址：

```json
{
  "mcpServers": {
    "文件服务器": {
      "command": "npx",
      "args": ["@anthropic/mcp-server-filesystem", "/Users/你的文档"]
    }
  }
}
```

### 第 3 步：让 Agent 用 MCP 干活（1 分钟）

```bash
# 现在你可以跟 Agent 说：
"帮我读昨天那个项目的 README 文件，然后根据里面的 API 文档，写一份接口调用示例"
```

Agent 会：通过 MCP 找到 README → 读取内容 → 理解 API 文档 → 写出调用示例。

**全程不需要你告诉 Agent "文件在哪里、怎么打开"。MCP 帮它搞定了。**

> **一句话：装 MCP 服务器 → 连上 Agent → 让它自己用。30 分钟，你的 Agent 从"只能说话"变成"能读一切"。**

## 4. MCP vs 传统 API：到底有什么区别？

**传统 API 是你写好代码让 Agent 调；MCP 是 Agent 自己发现工具、自己决定怎么用。**

| 维度 | 传统 API | MCP |
|-----|---------|-----|
| 对接方式 | 每个工具写一段代码 | 装一个服务器，自动对接 |
| 发现机制 | 你告诉 Agent 有什么工具 | Agent 自己查看可用工具列表 |
| 扩展性 | 加一个工具就要改代码 | 加一个 MCP 服务器就行 |
| 维护成本 | 高（每个工具单独维护） | 低（服务器统一管理） |
| 适用场景 | 固定、已知的工具 | 动态、不确定的工具需求 |

### 最适合用 MCP 的场景：

- 你的 Agent 需要访问 **多种工具**
- 你希望 Agent 能 **自己决定用什么工具**
- 你经常 **增删改工具**，不想每次改代码

> **一句话：如果你的 Agent 需要干活，用 MCP。如果只是简单问答，传统 API 就够了。**
